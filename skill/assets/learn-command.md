description: 최근 대화에서 발생한 실수/주의사항을 적절한 영역 가이드에 추가
---

사용자 입력: $ARGUMENTS

## 역할
최근 대화 컨텍스트를 분석해서 어느 영역의 가이드 파일에 무엇을 추가해야 할지 판단하고 반영한다.
대상 파일은 프로젝트 root의 map 파일(CLAUDE.md / AGENTS.md)이 가리키는 영역별 가이드 파일이다.

## 컨텍스트 범위
- "최근 대화"는 **현재 세션 내 사용자의 직전 메시지부터 시점까지**를 의미한다.
- 더 이전 메시지를 참조해야 한다고 판단되면 사용자에게 명시적으로 알린다.

## 절차

1. **메모 확정**
   - $ARGUMENTS가 있으면 그것을 추가할 내용으로 사용
   - 비어있으면 컨텍스트 범위 내에서 실수/금지 사항으로 보이는 내용을 추출

2. **영역 판단** (불명확 시 5단계에서 한 번에 확인)
   - 우선순위:
     1) 대화에 명시된 경로
     2) 최근 Edit/Write로 수정한 파일의 위치
     3) 키워드 매칭 (스택/도메인 용어)
   - 영역이 여러 개면 모두 후보로 둔다

3. **중복 체크**
   - 대상 가이드 파일의 "⚠️ LEARNED CAUTIONS" 섹션을 읽어서 동일/유사 내용이 이미 있는지 확인
   - 있으면 사용자에게 알리고 추가 여부 결정

4. **날짜 결정**
   - 시스템의 현재 날짜를 사용
   - 사용자가 다른 날짜를 명시한 경우 그 값 사용

5. **사용자 사전 확인 (통합)**
   - 추가할 내용 + 대상 가이드 경로(들) + 영역 판단 근거를 한 번에 보여주고 승인 요청
   - 영역 판단이 불명확했다면 이 단계에서 사용자가 보정

6. **반영**
   - 형식: `- (YYYY-MM-DD) <내용>`
   - "⚠️ LEARNED CAUTIONS" 섹션 하단에 추가
   - 섹션에 `_(아직 없음)_` 마커가 있으면 이를 첫 항목으로 교체
   - 영역이 여러 개면 모두에 추가

7. **커밋 안 함**
   - 파일만 수정. 커밋은 사용자가 다음 작업 시 함께 처리
   - 변경된 파일 경로만 출력

## 금지
- 사용자 사전 확인 없이 파일 수정
- 자동 커밋
- 영역이 불명확한데 임의로 추정해서 추가
