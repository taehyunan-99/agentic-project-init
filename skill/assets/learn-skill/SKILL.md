# learn — 영역 가이드에 학습된 주의사항 누적

## 입력 형태

| 호출 | 동작 |
|------|------|
| `/learn <메모>` (Codex는 `$learn <메모>`) | 사용자가 적은 메모를 그대로 누적 |
| `/learn` (인자 없음) | 최근 대화에서 잘못된 내용을 한 줄로 추출해 누적 |

## 절차

1. **메모 확정**
   - 인자가 있으면: 인자 텍스트를 그대로 메모로 사용 (윤색/요약 금지)
   - 인자가 없으면: 최근 대화 맥락에서 잘못된 내용을 한 줄 메모로 추출
     - 단서: 사용자의 정정 발화, 잘못된 가정으로 인한 실패, 코드만 봐선 알 수 없던 함정이 드러난 순간
   - 추출 결과를 사용자에게 보여주고 "이 내용으로 누적할까요?" 확인 후 진행

2. **현재 작업 영역 추론**
   - 직전 대화에서 다룬 파일 경로를 기반으로 영역 추론
   - root map 파일(`CLAUDE.md` 또는 `AGENTS.md`)의 "영역별 가이드" 섹션을 읽어 후보 영역 목록 확보
   - 추론이 모호하면 사용자에게 "어느 영역에 누적할까요?" 라고 물어 확정

3. **대상 가이드 파일 결정**
   - 영역 폴더에 `CLAUDE.md`만 있으면 → 그 파일에 누적
   - 영역 폴더에 `AGENTS.md`만 있으면 → 그 파일에 누적
   - 양쪽 다 있으면 (both 모드) → 양쪽 모두 누적 (sync hook이 commit 시 추가 보정)

4. **중복 체크**
   - 대상 가이드 파일의 "⚠️ LEARNED CAUTIONS" 섹션을 읽어 동일/유사 내용이 이미 있는지 확인
   - 있으면 사용자에게 알리고 추가 여부 결정

5. **누적 형식**
   - 형식: `- (YYYY-MM-DD) <메모>` (오늘 날짜 자동 삽입)
   - "⚠️ LEARNED CAUTIONS" 섹션 하단에 추가
   - 섹션이 비어있으면 (`_(아직 없음)_`) 그 줄을 첫 항목으로 교체
   - 섹션이 없으면 사용자에게 알리고 중단 (영역 가이드 템플릿이 깨졌다는 신호)

6. **확인**
   - 누적된 파일 경로 + 추가된 줄을 사용자에게 보여주고 마무리
   - 커밋은 자동으로 하지 않는다 (사용자가 직접)

## 원칙

- **인자 있는 호출은 메모 윤색 금지**: 사용자가 적은 표현 그대로 누적. 자기 표현이 나중에 검색/이해할 때 가장 정확함.
- **인자 없는 호출은 항상 사용자 확인 후 진행**: 추출한 메모를 보여주고 동의받은 다음에 누적.
- **영역이 모호하면 무조건 묻는다**: 잘못된 영역 누적이 가장 큰 실패 모드.

## 금지

- 사용자 사전 확인 없이 파일 수정
- 자동 커밋
- 영역이 불명확한데 임의로 추정해서 추가
