# 작업 체크리스트 — 가이드 작성/검증 전 점검 18항목

CLAUDE.md / AGENTS.md / 영역 가이드를 작성·수정하거나 스킬을 변경하기 전에 이 체크리스트로 점검한다. 컨텍스트가 길어질수록 원칙을 잊기 쉬우므로 매 구현 사이클의 입구로 사용한다.

> **Tradeoff**: 매 변경마다 18개를 손으로 점검하면 부담이 크다. 그래서 18개 중 15개는 `/guide-audit`가 자동 점검한다 (체크 도구 매핑 참고). 사람이 매번 봐야 할 항목은 16~18(카파시) 3개뿐.

---

## 사용 순서

1. **변경 범위 한 줄 명시** — 어떤 파일의 어떤 섹션을 어떻게 바꾸는가
2. **영향 받는 체크 항목 식별** — 18개 중 변경에 해당하는 것만 점검
3. **점검 — 통과 / 보정 / 보류 결정**
4. **구현**
5. **회귀 검증** — `/guide-audit`로 자체 템플릿 점수 변동 없음 확인

## @import 허용 케이스 (항목 13 상세)

| 케이스 | 위치 | 이유 |
|---|---|---|
| Both 모드 CLAUDE.md ↔ AGENTS.md | 모든 레벨 (root + 영역) | 에이전트 호환 sync. 추상화 없이 안 됨 |
| 영역 진입 후 강결합 의존 (API contract / schema SoT) | 영역 가이드 안에서만 | 영역 진입 후라 토큰 비용 부담 없음. 침묵의 가정 방지가 더 큼 |
| **(제외) root에서 영역 자동 import** | — | map의 토큰 절약 핵심 가치 보존 |

**"매우 강한 결합 vs 강한 결합"의 경계가 모호한 케이스는 사용자 판단으로 결정한다.** 자동 룰을 강제하지 않는다.
